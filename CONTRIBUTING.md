# Contributing to the SAE Framework

Thank you for helping build the future of software engineering! We are looking for community contributions in the following areas:
* **Tooling & Stacks:** Adding new "Zero-Ops" tools (frontend generation, instant backends, automated testing agents) to Module 4.
* **Case Studies:** Real-world examples of how you used AI to collapse the 12 layers and ship a production app solo.
* **Templates:** Markdown specification templates for Spec-Driven Development (SDD).

### Contribution Process
1. **Fork** the repository.
2. **Create a branch** for your feature (`git checkout -b feature/sae-module-upgrade`).
3. **Commit your changes** with clear, concise messages.
4. **Push to the branch** (`git push origin feature/sae-module-upgrade`).
5. **Open a Pull Request** against the `main` branch.

### Style Guide
* Keep concepts focused on **reducing human friction** and **maximizing AI context hygiene**.
* Avoid traditional project management jargon (e.g., trying to force standard Scrum/Agile ceremonies back into the framework).
