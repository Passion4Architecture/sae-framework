SAE FRAMEWORK
The Pain
If this is you, read on

You can generate code in seconds…
and trust it less than ever.

The AI made the typing free. It did not make the reviewing, the verifying, or the being-accountable free. For most teams, those got harder.

↑ PR review time

More generated code per change means more to scrutinize — review is now the slow part.

↓ Trust in AI output

Most developers report they don't trust AI output without checking it. Hallucinated correctness is the new defect class.

↑ Code churn

More code written, more thrown away. Speed without discipline is just faster rework.

The real problem

Faster generation collided with the same old human process — and the hand‑offs between people became the bottleneck. We need an operating model built for this constraint, not the last one.

Figures are directional industry signals (developer-sentiment surveys, code-churn analyses), not precise universal measurements — see the wiki's Verification Doctrine for sourcing.
SAE FRAMEWORK
The Axiom
One idea — everything else is a corollary

When the cost of generating code approaches zero, the binding constraint becomes clarity.

So we re‑base the operating model on clarity, not effort — the unit of work, the source of truth, the quality gate, and the metrics all change.

SAE FRAMEWORK
Precision
Be precise — or the skeptics win

What collapsed, and what did not

Collapsed → ~0

Syntax cost. Writing the code, the boilerplate, the scaffolding, the CRUD. The thing every prior era spent its scarcity budget on.

Did NOT collapse

Clarity. Verification. Accountability. Knowing what to build, proving it's correct, and being answerable for it. These got relatively more expensive.

Pre‑empting Brooks

"No Silver Bullet" was right: coding was never the only bottleneck. SAE does not claim execution collapsed — only that syntax did, while the hard parts remained. That's exactly why the framework elevates verification and clarity. Brooks sharpens the thesis; he doesn't refute it.

Context Entropy — measure preparation, not effort

Agile measures effort (story points). Vibe coding measures nothing. SAE measures how hard the intent is to pin down before generation — the real labor when syntax is free.

SAE FRAMEWORK
Signature Idea · 2
The other moat

The writer never judges its own work

Almost all AI coding has the same model in the same context both write and review its output — exactly where hallucinated correctness survives.

Self‑review

The author can't see the gap between what it meant and what it wrote. Confirmation, not verification.

Adversarial Independence

A fresh‑context Guardian sees the spec and interfaces (grey‑box) and is tasked to refute, not approve.

Reality check

Independence cuts correlated error — it doesn't eliminate it. A wrong spec makes Engineer and Guardian agree, and both are wrong. No fresh context fixes a flawed contract.

SAE FRAMEWORK
The Spine
Built on prior art, sharpened

Spec‑as‑truth, with measured drift · blast‑radius gating

The spec is the source of truth

Fix bugs by editing the spec first, then regenerating. The delta over Spec Kit / Kiro: we measure the drift between code and spec — spec‑drift time is the decay canary.

Gate the irreversible by blast radius

Entropy = how hard to prep. Blast radius = what breaks if wrong. A 1‑line auth change is low entropy, high blast radius — auth, payments, deletion, schema drops are never auto‑applied.

Honest about lineage

Spec‑driven development is prior art (GitHub Spec Kit, AWS Kiro). SAE's originality is the instrumentation — measured drift and structural adversarial independence — not the concept.

SAE FRAMEWORK
Operating Model
End‑to‑end lifecycle

One spec‑centric loop, every change

Every fix re‑enters at Spec

Never patch the code and forget the spec — that's the #1 failure mode (a stale spec). Telemetry → spec amendment → one‑click approve → ship.

Two gates never relax

Reconcile the spec after every change; keep a human gate on the irreversible (auth · payments · deletion · schema). Everything else scales down for trivial work.

Credibility
A first‑class feature, not a disclaimer

Honest by construction

Every bold claim ships paired with a named Reality Check. We steel‑man the objections before answering them.

"Vibe coding with a manifesto"

The opposite — SAE adds the discipline vibe coding lacks: independence, measured drift, gating.

"O(1) is hype"

Conceded — O(1) is coordination cost only; cognitive cost is conserved as accountability.

"No evidence"

True today. SAE is a falsifiable hypothesis with instrumentation — the KPI kit is the proof engine.

"Can't survive regulation"

Keep the mandated gates; apply SAE within them to remove latency, not controls.

A framework that can't name its own limits is marketing. The full steel‑man set lives on the wiki's Objections page.
